Nomad — Design Specification
Complete design spec for both the initial prototype (V1 — for immediate development) and the enhanced version (V2 — for future iteration). All tokens, components, screens, and interactions are documented here.
For developers: Start with V1 screens — these represent the minimum viable product scope. V2 screens show future enhancements to be built after V1 ships. Use the token values in this document as your single source of truth.

Figma Setup Guide
How to get these designs into Figma in under 15 minutes
Fastest method: Use the "HTML to Figma" plugin (free, by ANIMA). Open Figma → Plugins → HTML to Figma → paste the prototype URL or file → it converts every screen into frames with auto-layout.
StepActionDetail
1. Create fileNew Figma file, name it "Nomad — V1 Prototype"Set canvas to Mobile (390×844)
2. Install fontsInstall Syne and DM Sans from Google FontsOr use Figma's built-in font picker — both are available
3. Create pagesPage 1: Design Tokens · Page 2: V1 Screens · Page 3: V2 Screens · Page 4: Components · Page 5: User FlowMatch the sidebar structure of this doc
4. Set up tokensCreate Colour Styles from the palette belowName format: Palette/Primary, Palette/Accent, etc.
5. Import screensUse HTML to Figma plugin OR manually recreate each frameFrame size: 390×844px for each screen
6. Add connectionsPrototype tab → connect screens per the user flowUse the flow diagram in this doc
7. AnnotateAdd spacing/size annotations using the Figma Measure pluginAll measurements in this doc are in px
8. ShareShare → "Anyone with link can view" → send to devsDevs use Inspect panel for exact values
